# {{PROJECT_NAME}} docs

Modeled after the StackAlchemist docs skeleton.

## Layout

- `product/` — PRD, PDD, BRD, use cases. What we're building and why.
- `architecture/` — SDD, data-flow, ERD, testing strategy. How it's built.
- `user/` — getting-started, guides, FAQ, troubleshooting. For humans using the service.
- `advanced-docs/` — self-hosting, ops patterns, non-default deployments.
- `runbooks/` — incident response, rollback, on-call playbook.
- `branding/` — visual guidelines (if user-facing).

Keep each file terse. Diagrams as mermaid/PlantUML in-file, not external PNGs.
